# 大学主题 PPT Skill 工作流契约

本文件定义 skill 在面对大学主题 PPT 请求时的执行流程。它只描述流程和决策，不描述具体脚本实现。

## 稳定执行原则

任何生成或迁移任务都必须先落到 `deck_spec.json`，再进入 PPTX 组装。

推荐命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\new_deck_spec.ps1 -SkillRoot . -SchoolId ruc -Purpose thesis_defense -TargetSlideCount 12 -OutputPath .\outputs\ruc_thesis\deck_spec.json
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate_deck_spec.ps1 -SkillRoot . -SpecPath .\outputs\ruc_thesis\deck_spec.json -Strict
```

如果 spec 校验不通过，不得继续生成正式 PPTX。先修复资产、schema、页面计划或主题 token。

## 设计目标

- 输出优先级：可编辑 PPTX > 预览图 > 其他格式。
- 核心差异化：将通用信息结构版式套入具体大学身份系统，而不是只贴 logo。
- 质量标准：每页要有明确的信息关系、统一的学校主题、可编辑对象、合格的视觉密度。
- 风险控制：颜色、logo、学校文字、图片、字体、母版、旧模板残留必须自动检查。

## 请求类型

先判断用户请求属于哪一种模式：

| 模式 | 触发语义 | 输出 |
|---|---|---|
| `new_deck` | 做一套某大学 PPT、根据主题生成 PPT、毕业答辩/课程汇报/项目路演 | 新 PPTX + 预览图 |
| `theme_migration` | 把 A 大学 PPT 改成 B 大学、换成某校风格 | 迁移后的 PPTX + 预览图 |
| `template_library` | 增加某校模板、复刻模板、整理版式库 | 结构套件/内容版式库文件 |
| `edit_existing` | 改第几页、删某页、替换 logo/图片/文字 | 修改后的 PPTX + 预览图 |
| `asset_onboarding` | 增加学校资产、找 logo/配色/校园图 | 学校资产包 + 待确认清单 |

不生成 PPTX 的讨论、选题、文案润色，不触发本 skill，除非用户明确要求制作或修改 PPT 文件。

## 总流程

1. 解析请求：提取学校、用途、页数、语言、是否已有内容、是否有源 PPTX。
2. 路由模式：选择 `new_deck`、`theme_migration`、`template_library`、`edit_existing` 或 `asset_onboarding`。
3. 查学校资产库：读取目标学校 `brand.json` 和资产文件。
4. 若学校不存在：进入学校资产补全流程。
5. 生成或读取内容大纲：形成 `slides_plan.md`，必要时给用户确认。
6. 生成 `deck_spec.json`：锁定学校、结构套件、色彩映射、内容页母版、版式选择和校验规则。
7. 严格校验 spec：运行 `validate_deck_spec.ps1 -Strict`。
8. 预览风格：默认给 3 套四件套 + 内容母版预览，用户选择后再完整生成。
9. 组装 PPTX：使用结构套件、内容母版和内容版式库生成可编辑 PPTX。
10. 自动校验：渲染预览、检查颜色、logo、字体、旧文字、版式越界和图片来源。
11. 交付：输出 PPTX、预览图、校验摘要和可修改建议。

## 用户信息收集

优先从用户原话和附件推断。只在缺口影响结果时追问。

必须明确的信息：

- 学校：如中国人民大学、复旦大学、南京林业大学。
- 用途：毕业答辩、课程汇报、项目路演、学术报告、社团活动等。
- 内容状态：已有完整内容、只有大纲、只有主题。
- 页数范围：短 5-10、中 10-20、长 20+。
- 图片策略：默认少用图片；只有学校身份图、用户提供图或必要图文页使用图片。

可自动默认的信息：

- 输出格式：PPTX。
- 比例：16:9。
- 中文字体：微软雅黑。
- 结构：封面、目录、章节页、内容页、结尾页。
- 每类内容版式候选：2-3 个。

## 学校资产流程

读取目标学校资产目录：

```text
assets/schools/<school_id>/
  brand.json
  logo/
  photos/
  source-notes.md
```

如果资产完整：

1. 使用 `brand.json` 中的主色、深色、浅色、辅助色。
2. 使用彩色 logo 于浅色背景，白色 logo 于深色背景。
3. 使用学校图片时，只从该学校已确认图片池中选。

如果资产缺失：

1. 告诉用户缺少哪些资产：logo、白色 logo、主色、辅助色、校园图。
2. 若允许联网检索，检索后只产出候选，不直接认定。
3. 用户确认后才写入资产库。

## 视觉预览流程

学习 Zara `frontend-slides` 的做法：让用户看真实视觉选项，而不是只描述风格。

对 `new_deck` 默认生成 3 套预览：

| 选项 | 典型方向 | 适用 |
|---|---|---|
| A | 学术答辩型 | 毕业论文、开题报告、课程论文 |
| B | 汇报展示型 | 项目汇报、团队展示、竞赛路演 |
| C | 简约正式型 | 课程展示、学院汇报、正式会议 |

每套预览至少包含：

- 封面
- 目录
- 章节页
- 内容页母版示例
- 结尾页

用户选择后，把所选套件写入 `deck_spec.json.structure_suite`。如果用户要求混合选项，记录每个页面类型对应的来源套件。

## 内容规划流程

`slides_plan.md` 是用户和 skill 共同确认的内容源。

格式：

```markdown
---
title: 此处填写 PPT 标题
school: 中国人民大学
purpose: 毕业论文答辩
target_slides: 16
---

## 1. [cover] 此处填写标题
副标题：此处填写副标题

## 2. [toc] 目录
01 标题
02 标题
03 标题

## 3. [section] 标题

## 4. [content, relation=process] 标题
- 此处填写文字
- 此处填写文字
```

规则：

- 每页必须有页面类型。
- 内容页必须标注 `relation`，如 `process`、`timeline`、`comparison`。
- 用户无内容时，用无信息占位文本，不编造具体事实。
- 用户有内容时，保留含义，但改写成适合 PPT 的短句。

## 版式选择流程

内容页版式由信息关系决定，而不是随机挑选。

1. 读取每页 `relation`。
2. 从 `layout_taxonomy.md` 中找到关系类型。
3. 在内容版式库中挑 2-3 个候选。
4. 依据文本量、图片量、是否数据页、是否人物页、是否流程页选择最终版式。
5. 连续页面不得重复同一具体版式，除非用户明确要求统一。

## 结构套件和内容版式的边界

结构套件负责：

- 封面
- 目录
- 章节页
- 内容页母版边框
- 页眉/页脚/导航/页码
- 结尾页

内容版式负责：

- 页面中间的信息结构
- 卡片、流程、图表、对比、人物、图文、时间线等中心对象

内容版式不得自带学校 logo、页眉、页脚、导航条和来源文字。中心版式只保留必要占位文字。

## 主题迁移流程

从 A 学校迁移到 B 学校时：

1. 保持所有版式位置和形状结构不变。
2. 主色映射到主色，深色映射到深色，浅色映射到浅色。
3. 强附色只映射到目标学校允许的辅助色。
4. 替换 logo、校名、英文名、校训、校园图。
5. 检查 slides、masters、layouts、charts、groups、gradients、shadows。

禁止：

- 因换学校而简化复杂形状。
- 把封面/章节页改成满屏主色。
- 拉伸 logo。
- 留下源学校文字、图片或颜色残留。

## 图片策略

默认少用图片。优先使用纯图形、学校 logo、色块、线条和可编辑对象。

允许使用图片的场景：

- 封面或结尾需要弱化的学校代表性背景图。
- 图文页本身需要图片。
- 用户提供了明确图片。
- 学校身份弱，需要通过校园图增强识别。

不允许使用图片的场景：

- 非目标学校图片。
- 来源不明且未经确认的网络图片。
- 封面目录大量依赖图片导致后续泛化困难。
- 将整页文字和图表做成不可编辑图片。

## 自动校验流程

每次交付前必须执行：

1. 渲染整套 PPTX 为预览图或 contact sheet。
2. 检查所有 slide 尺寸为 16:9。
3. 检查 logo 宽高比未被拉伸。
4. 检查主色和辅助色没有越界。
5. 检查旧学校文字、模板来源文字、作者水印残留。
6. 检查字体是否统一为微软雅黑。
7. 检查封面、目录、章节、结尾是否符合结构套件规则。
8. 检查内容页是否有多余文本框和无意义来源文字。
9. 检查图片是否属于目标学校或用户提供素材。
10. 检查预览中是否有明显遮挡、错位、空白页、4:3 留白。

校验失败时先修复，再重新渲染预览。

## 工作流回归测试

每次修改 skill 后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\check_assets.ps1 -SkillRoot .
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_workflow_tests.ps1 -SkillRoot .
```

回归测试至少覆盖：

- 已入库红色系样本学校新建 deck spec。
- 已入库蓝色系学校新建 deck spec。
- 已入库绿色系学校新建 deck spec。
- 未入库学校应失败并进入资产补全。

更多测试说明见 `references/test_protocol.md`。

## 交付格式

交付时给用户：

- PPTX 文件路径。
- 预览图路径。
- 使用的学校主题。
- 使用的结构套件。
- 页数。
- 校验摘要。
- 可继续修改的方向。

不要把内部 JSON、脚本参数、缓存路径暴露给普通用户，除非用户询问技术细节。
